# Final Result Post and Reflections

By the end of the developing process, the game is in the following state:
* An initial menu scene welcomes the player with play and options buttons.
* The player can freely move around the map, shooting enemies and collecting ammo and health objects.
* Enemies have basic AI that allow them to shoot the player, move around and change animations depending on state.
* Only one map, one kind of gun and one kind of enemy was added

## Conclusion

There is still a lot to be implemented and expanded in Wind of Change. 
For instance, a better AI would definitely improve the game. This AI could use the environment to cover from the player’s shots or follow the player around the map.
Wind of Change is also lacking a lot of content (if it wanted to be an actually enjoyable game), such as a better map design, together with more maps/levels, different types of weapons, like pistols, rocket launcher or melee weapons, new enemies that would fit the different parts of the map, as well as a more narrative design.
All of these ideas were left behind due to a lack of time, and definitely offer a big window for future development of the game.

Nonetheless, the final result is a (short and simple but) completely playable game, considering the aspects mentioned above. 
When the game was done and ready to be tested out, I realized there were some issues realted to the input system. Since the FPS controller asset was using the old input system, when trying to adapt to the new one, using the file provided by the teacher, inputs would work but not with the correct binds. After trying to solve this issue using both the old and the new system, little progress was made. This resulted in the game only working for keboard and mouse and partially in controller (with broken binds)
